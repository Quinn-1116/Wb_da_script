# 分析框架
* 想要回答的核心问题是：最优的开局人数是几人？如何定义最优？！
    - 4人局、5人局...10人局每种类型对局的每日占比？（现状）
    - 有没有用户（房主）对某种人数的对局有所偏好？如始终限制一定房间人数再开局。理论上不太可能，因为，即便房主点了「开始游戏」，房间内仍然会涌入玩家。所以房间内的人数是一个随机变量，但即便如此，也能看出每个用户每天不同类型的对局参加了几局。假如一个玩家玩过的10局游戏中，超过6局都是4人局，就可以将其定义为4人局用户。
    - 弄清楚上述问题，看一下每天的，4人局、5人局...10人局用户有多少？（按新老用户进行拆分），这里可能会出现某一种类型的用户群数量特别小的情况，倘若如此，考虑把数量小的对局类型接近的（这里说的接近是指对局人数的接近）几个用户群合为一个群体。
    - 对上述群体展开不限于以下的分析：
        + 他们分别是什么样的人？（用户画像）
            * 游戏分场选择，新手or进阶？
            * 年龄分场选择，小萝莉or老阿姨？
            * 游戏等级分布（11.30日开始有的）
        + 黏性
            + 新用户->留存,老用户->回访;至少要看R2和R7
        + 游戏活跃度
            * 游戏局数,游戏时长;
        + 付费
            * 付费比例,arpu,arppu;
